# yc-ad — 物件廣告全流程

## 何時使用

使用者提到任何物件編號（格式 `YC` + 7 碼數字，例 `YC1835328`）並有以下意圖之一：
- 要產文案 / 準備發廣告 / 發某個社團
- 要把物件同步上 KEIS 廣告追蹤
- 要存粉專貼文連結
- 要登錄社團發文紀錄
- 要撤除廣告

不確定意圖時，假設「走全流程」（先產文案）。

## 主流程：使用者說「發 YCxxx」或類似

### Step 1. 查 Notion 物件

用 `notion-search` 或直接 query data source，篩選 `案件編號 = YCxxx`，拿到該 page 的 ID 與所有欄位資料。

若查無此物件 → 提示使用者：「Notion 裡找不到 YCxxx。物件是由『廣告v3 掃描發文線』每天自動掃 KEIS 建檔的，查無代表這案還沒被掃到或不在自店白名單內。」停。
（LINE 的「建檔 永慶網址」指令已於 2026-07-23 退役）

### Step 2. 產兩版文案

依下面規格產出，**不要套用統一模板**（每次微調開頭和排版避免被 FB 判洗版）：

#### 粉專詳細版（控制在 200–300 字）

結構：
1. 第一行：吸睛標題，含地段 + 房型 + 賣點 + 價格。使用「【】」框起。
2. 物件規格區（用 emoji 開頭，每行一個）：📍地址、🏠格局、📐坪數、🏢屋齡+樓層、💰開價
3. 空一行
4. ✨ 物件亮點區：3-4 條，每條一行，用「・」開頭。素材來源＝Notion 既有欄位（社區名稱／建物類型／格局／樓層／屋齡／坪數）＋ 永慶官網連結上的公開描述。**Notion 沒有的事實不准寫**（原「特色說明」欄已於 2026-07-23 刪除，v3 不再提供原始描述）
5. 空一行
6. 聯絡資訊（用「聯絡資訊 / 經紀業」段落的固定資料）：
   ```
   📞 薛先生 0912877583（同 LINE）
   LINE：https://line.me/ti/p/kg1pMk4vX8
   ```
7. 空一行 + 分隔符 `—`
8. 經紀業資訊（法規必填，每篇都要帶）：
   ```
   凱璿誼峰不動產有限公司
   經紀業特許字號：高市地政籍字第11034791100號
   證照字號：（111）登字第417208號
   ```

**規則**：
- 不要寫成 AI 通用文案口吻（「歡迎您」「快來看看」之類官腔避免）
- 用台灣房仲的真實口吻
- 不要塞外部連結到內文（LINE 連結算聯絡資訊，例外允許）
- 不要捏造資料：屋齡/坪數/格局必須用 Notion 實際值
- 若 Notion 缺欄位資料，那行省略，不要寫「未提供」
- **不要**在文末加 `#YCxxxxxxx` hashtag（使用者偏好：不顯示物件編號）

#### 社團簡短版（內文控制在 50–80 字，下方再加固定經紀業 footer）

結構：
1. 一行吸睛標題：用 emoji 開頭，地段 + 房型 + 1 個關鍵賣點
2. 1-2 行重點賣點（學區 / 交通 / 屋齡 / 含車位 ... 挑最強的 1-2 個）
3. 引導語：「詳情看留言」或「私訊索取完整資料」（兩種輪流用，每次不一樣）
4. 空一行 + 分隔符 `—`
5. 固定經紀業 footer（法規必填，字數不算進 50-80 限制）：
   ```
   凱璿誼峰不動產有限公司｜薛先生 0912877583
   經紀業特許字號：高市地政籍字第11034791100號
   ```

**規則**：
- 內文絕對不要放粉專或外部連結（連結放留言區是 SOP 規矩；footer 的經紀業資訊不算）
- 不要寫具體價格（吊胃口）或只寫概略「2字頭」「3字頭」
- 每次叫這個 skill 重產社團版時，開頭 emoji、結構、用詞要換一套，避免被判定為複製貼上
- 經紀業 footer 每次都要帶

#### 寫回 Notion

產完兩版後，用 Notion MCP PATCH 該 page：
- `粉專文案` ← 粉專詳細版
- `社團文案` ← 社團簡短版
- `文案版本` ← 既有值 + 1

### Step 3. 呈現給使用者

兩段文案分開貼出，每段用 ```` ``` ```` 包起來方便整段複製。格式：

```
✅ 已產出 YCxxxxxxx 兩版文案（已存進 Notion，文案版本 = N）

【粉專詳細版】複製整段貼到粉專
---
（粉專文案）
---

【社團簡短版】複製整段貼到社團（每篇社團發前再叫我重產一版，避免洗版判定）
---
（社團文案）
---

接下來：
1. 把粉專版貼到粉專、發布 → 點時間戳取得永久連結 → 回我「粉專連結是 https://...」
2. 要我幫你同步上 KEIS 嗎？回「同步 KEIS」
3. 發到社團後回我「發到 X 社團了」
```

### Step 4. 接收後續輸入並分別處理

之後使用者每講一句，依關鍵字判斷：

#### 4a. 使用者貼粉專連結（「粉專連結是 https://www.facebook.com/...」）

- PATCH Notion 該物件 `粉專貼文連結` 欄位
- 同時把 `狀態` 改為 `已發布`
- 回：「✓ 粉專連結已存。要同步 KEIS 嗎？發到哪個社團了嗎？」

#### 4b. 使用者說「同步 KEIS」

從 Notion 抓該物件的 `來源連結`（永慶網址，格式 `https://buy.yungching.com.tw/house/{7位數字}`，**不是** YC 編號）和 `粉專貼文連結`，然後輸出下列「Claude 擴充功能操作指令包」給使用者複製：

```
步驟：

前往 https://keis.kshouse.com.tw/ad-tracker
點右上角「+ 新增廣告」按鈕
在「官網連結」欄貼上：{{來源連結}}
按「自動填入」→ 等待 3 秒讓系統抓取標題、地址、價格
若出現「已新增過此連結」警告 → 停止並告知使用者
在「廣告平台」勾選「臉書」checkbox
在出現的「臉書連結」欄貼上（必須是 Facebook 貼文完整網址，不要用分享按鈕的短網址）：{{粉專貼文連結}}
在「備忘錄」欄填入：{{案件編號}}
按「確定新增」
確認列表出現「新上架」標籤即完成
```

把 `{{...}}` 用實際值代入。

提示使用者：「貼給你的 Claude 瀏覽器擴充功能執行。完成後跟我說『KEIS 上架成功』，我把 Notion 標已同步。如果擴充功能回報『已新增過此連結』，代表這筆我之前同步過了，跟我說一聲我也標已同步。」

#### 4c. 使用者說「KEIS 上架成功」/「KEIS 完成」/「KEIS 之前已新增過」

PATCH Notion `KEIS同步` ← `已同步`。

- 若是新上架成功：回「✓ 已標 KEIS 同步完成」
- 若是擴充功能回報「已新增過此連結」：回「✓ 這筆之前就同步過了，Notion 也標起來。要查既有那筆可以到 KEIS 廣告追蹤頁搜尋此編號 `{{案件編號}}`」

#### 4d. 使用者說「發到 X 社團」「貼到 Y 社」「X 社團發了」

從 Notion 抓 `廣告貼文紀錄` 既有值，append 一行：
```
社團名 / YYYY-MM-DD
```
（日期用今天）

中間用換行分隔（不要分號或逗號）。PATCH 回 Notion。回：「✓ 已記錄『X 社團』。SOP 建議：兩篇社團之間隔 30 分鐘以上、一天總共 5-8 個社團以內、避免短時間連續發。下次要發前再叫我重產一版社團文案。」

如果使用者一次講多個社團（「發到 A、B、C 了」），全部 append。

#### 4e. 使用者說「重產社團文案」或「再給我一版」

只重產社團簡短版（不動粉專版），換不同開頭/排版。PATCH 回 `社團文案` 欄位 + `文案版本` +1。

#### 4f. 使用者說「YCxxx 已撤除」或「下架」

- PATCH Notion：`狀態` ← `下架`、`已撤除確認` ← true、`下架偵測時間` ← 今天
- 若 Notion `粉專貼文連結` 有值，回覆訊息附上該連結：
  ```
  ✓ Notion 已標下架。
  請手動刪除粉專那篇貼文（社團那邊的連結會自動失效）：
  {{粉專貼文連結}}
  ```
- 提示：刪了粉專貼文後，所有社團裡的留言連結會變「內容無法使用」，不用回頭刪社團貼文。

## 替代入口

### 使用者直接說「同步 YCxxx 到 KEIS」（跳過產文案）

如果 Notion 該物件已有 `粉專貼文連結`，直接跳 Step 4b 給操作包。
若沒有，提示：「先發粉專貼文取得永久連結，回我『粉專連結是 ...』之後再同步」。

### 使用者只說「YCxxx」不講要做什麼

簡短列當前該物件狀態（文案有沒有產過、粉專連結有沒有存、KEIS 同步沒、發過哪些社團），然後問下一步要做什麼。

## 規矩（絕對不要違反）

- ❌ 不要自動操作 FB（不發、不刪、不留言）。FB 操作一律由使用者手動完成，本 skill 只負責產內容與紀錄
- ❌ 不要自動操作 KEIS。給操作指令包讓使用者貼給 Claude 瀏覽器擴充功能執行
- ❌ 不要捏造物件資料（坪數、屋齡、實價登錄參考）。Notion 沒有就省略
- ❌ 不要把粉專連結放進社團簡短版內文（SOP：連結放留言區）
- ❌ 不要在貼文範本裡寫「轉貼自...」「@提及人名」這類會被 FB 判垃圾的東西
