# 不動產售屋表自動填寫工具

> 狀態：🟢 全流程已實測通過（左營／三民多筆）。桌面 GUI 工具，與 n8n 無關、獨立執行。
> 未驗證的座標：工業區 K42「乙種工」寫法、車位多層細項（地上/地下、平面/機械、上中下橫移、入口）——需拿對應案件實際開 Excel 確認。

謄本 PDF → 自動查 104 社區資料 + 高雄市使用分區 → 跳確認視窗逐項核對 → 產出售屋表 Excel。

## 檔案
- `gui_main.py`：主程式（tkinter GUI）+ 填表邏輯（`fill_excel`）
- `bot_104.py`：Selenium 駕駛 104woo（自動登入、搜社區）+ 高雄市使用分區查詢（`fetch_zoning`）
- `confirm_wizard.py`：開始產出前的逐項確認視窗
- `parser.py`：謄本 PDF 解析（`parse_land` / `parse_building` / `merge`）——2026-07-15 起改進版控（之前只在本機，改壞了沒有版本可以回頭救）

## 執行
```
python gui_main.py
```
（Windows 可雙擊 `啟動工具.vbs`）

## 第一次設定
```
cp .env.example .env
# 編輯 .env 填入 104woo 登入帳密（WOO104_ACCOUNT / WOO104_PASSWORD）
```

## 本機相依（未進版控，在使用者本機 zip 內）
- `template/sale_template.xltx`：售屋表 Excel 範本（二進位檔，故意不進 git）
- `output/`：產出資料夾
- 套件：`selenium`、`openpyxl`、`python-dotenv`、Chrome + chromedriver

## 流程
1. 選土地 + 建物謄本 PDF
2. 「開啟 104（自動登入）」→ 自動帶入帳密登入
3. 「完成登入，自動產出」→ 解析謄本 → 搜 104 社區 → 查使用分區 → 確認視窗 → 產出
   （或不用 104，直接「開始產出售屋表」走純謄本路徑）

## 使用分區填表規則
- 住宅區 → 勾 B42（黃）+ E42 填種別國字（三、四…）
- 商業區 → 勾 B44（黃）+ E44 填種別國字
- 工業區 → 勾 G42（黃）+ K42 填「乙種工」之類
- 其他（市場用地等）→ G44 填整串文字並塗黃

## 注意
- **確認 wizard 改左右箭頭導航**（2026-07-17 修）：`confirm_wizard.py` 每個確認視窗左右各一個 ◀/▶ 箭頭做上一步/下一步（第 1 步 ◀ 反灰），底部按鈕列只留動作鈕（一般視窗剩「取消」；訴求重點視窗剩「先不填/填寫完畢/取消」）。移除「略過」按鈕，效果由「欄位清空按 ▶」完整取代。◀ 回退時會靜默存檔已輸入的內容，數字格式打到一半也不擋回退（該欄維持回退前原值）。訴求重點「先不填」會清空全部訴求重點欄位（不只當欄），「填寫完畢」則保留其餘已填欄位。勾選型視窗現在也可以不勾直接按 ▶/Enter 留空跳過。
- **公設分算逐建號計算**（2026-07-17 修）：共有部分改成從「共有部分」關鍵字逐建號切塊解析（`common_parts`），每筆 面積 × 權利範圍 加總；舊版只認一種固定行序格式，遇到「8945.29 m² × 100000分之345」這類版面直接整段漏抓、公設變空白。車位持分（「含停車位…權利範圍…」）單獨拆出：車位坪填 `F28`、其餘公設填 `F30`，**F28 + F30 = 謄本共有部分權利範圍換算合計**。解析後 log 會印出每筆共有部分與分算結果供核對。

- **建物售屋表訴求重點 wizard 補回**（2026-07-17 修）：`AL30/32/34/36/38` 共 5 欄，wizard 最後逐欄跳視窗（先不填 / 填下一個 / 填寫完畢），之前只有土地表有這段、建物表整段不見。
- **無車位也要塗 AN16**（2026-07-17 修）：wizard 車位答「無」時塗黃 `AN16`（車位:無 方格），舊版什麼都不塗。

- **`fill_excel()` 存檔前必須 `wb.template = False`**：範本是 `.xltx`，openpyxl 讀進來會記住範本旗標，不關掉存出的 `.xlsx` 內部類型會是 `template.main+xml`，嚴格版 Excel（別台電腦）直接拒開。已修（git + 本機兩份都改）。
- **主建物坪數依謄本逐層列出**（2026-07-15 修）：`parse_building()` 舊版用 `re.search` 只抓第一筆「層次 X 層次面積 Y」，透天等多層建物的 2 樓以上、突出物全部漏掉，整棟坪數擠進 `F12`「室內」一格，`F14/F16/F18/F20`（範本的 4 個「F」列）永遠空白。改用 `re.finditer` 抓全部層次，`fill_excel()` 依序分別填進這 5 格；超過 5 筆（範本格數上限）會合併進最後一格並在 log 提醒人工核對。附屬建物若出現非「陽台/雨遮/花台」的類型，會另外填進「其他-」欄（`C26`/`F26`），不再默默漏掉。
- **入口型式（車位）wizard 步驟已補 show 條件**（2026-07-15 修）：`confirm_wizard.py` 車位「有無」答「無」後會跳過車位細節（位置/樓層/編號/類型/機械層位），但舊版「入口型式」（坡道式/機械升降式，本來就是車位入口的敘述）沒有這個條件，沒車位還是會被問、答了還會填進表格，邏輯矛盾。已補 `show: _parking_yn == '有'`。
- **主建物樓層名稱寫 C 欄**（2026-07-15 修，同日二修）：逐層列出坪數的同時，層次名稱（一層/二層/三層/四層/突出物一層…，`_clean_floor_label()` 去掉謄本「突出物一層：２５．７２平方公尺」格式裡冒號後面重複的面積字樣）寫進 **C12/C14/C16/C18/C20**（`C12:D13` 合併格的頂格）。⚠️ 12~20 列的標籤區是 `C:D` + `E` 兩個**相鄰的合併儲存格**：E 欄（`E12:E13`…）是範本預設「室內」/「F」標籤、要保持原樣不動，第一版誤寫 E 欄結果蓋掉範本標籤且視覺偏右一格。只有真的有多層（`floor_pings` 筆數 >1）才會寫，一般大樓單一樓層戶 C 欄維持空白。
- **多筆土地謄本加總後，所有權判斷沒有跟著更新**（2026-07-15 修）：`_combine_parcels()`／`_build_land_data()` 舊版只加總地坪坪數，沒有同步加總持分比例——3 個共有人謄本都餵了、地坪也加對了（如 3分之1×3=全部），但 `所有權` 欄位還是照第 1 筆謄本標「持分 3分之1」（填 `B38`），沒有改標「全部」（`B36`）。已加持分加總（誤差 <0.01 視為等於 1）判斷，加總≈全部就改標 `B36`。
- **地上權支援**（2026-07-15 加）：判斷依據是「有沒有選土地謄本」——建物案流程改成只有建物謄本是必填，土地謄本可以不選；`_combine_parcels(land_paths=[], ...)` 偵測到沒選土地就直接把 `ownership` 標成 `'地上權'`，`fill_excel()` 對應勾 `G38`（地上權，`B36`/`B38`/`H38`/地坪都保持空白）。「開啟104」也同步放寬，104 查詢本來就只需要建物門牌，不需要土地謄本。
