# FB 永久金鑰設定教學（步驟 1 專用）

目標：拿到一把**永不過期**的粉專金鑰，讓 n8n 可以自動發文、刪文。走「Meta 企業管理平台 → 系統使用者」路線，全程**不會**碰到之前卡死兩次的網址白名單問題（FBL4B，歷史看 git log）。

> 給帶流程的 Claude：Meta 後台介面常改版，按鈕名稱可能跟下面寫的略有出入。每一步都附了「這步的目的」，找不到按鈕時照目的找同義的入口。使用者卡住時請他截圖。

**前置條件**：使用者的 FB 個人帳號是粉專的管理員。全程用他的帳號登入操作。

---

## A. 企業管理平台（沒有就建，有就跳過）

**目的**：系統使用者只能活在企業管理平台裡，先有這個容器。

1. 開 `https://business.facebook.com`
2. 沒有企業帳號 → 「建立帳號」：企業名稱填「凱璿誼峰不動產有限公司」、姓名、公司 email
3. 建好後進「設定」（左下齒輪）

## B. 把粉專加進企業管理平台

**目的**：粉專變成企業資產，之後才能指派給系統使用者。

1. 設定 → 「帳號」→「粉絲專頁」→「新增」→「新增粉絲專頁」
2. 輸入粉專名稱選取 → 因為他本人就是管理員，會直接通過，不用等審核

## C. 建一個 App

**目的**：金鑰一定要掛在某個 App 底下，建一個空殼商用 App。

1. 開 `https://developers.facebook.com` → 「我的應用程式」→「建立應用程式」
2. 類型／用途選「**商家**」（Business）；名稱隨意，例 `kaixuan-ad-bot`
3. 建立時「企業管理平台帳號」欄位選剛才的企業帳號（把 App 綁進去）
4. App **停留在開發模式即可**，不用送審、不用上線——發自己企業資產（粉專）的文不需要審核

## D. 建系統使用者

**目的**：一個不會過期、不會被登出的機器人身分。

1. 回 `business.facebook.com` → 設定 → 「使用者」→「系統使用者」→「新增」
2. 名稱例 `n8n-bot`，角色選「**管理員**」

## E. 把資產指派給系統使用者

**目的**：讓機器人身分有權操作粉專和 App。

1. 點進剛建的系統使用者 → 「新增資產」
2. 「粉絲專頁」→ 選粉專 → 開「**完整控制**」→ 儲存
3. 再一次「新增資產」→「應用程式」→ 選 `kaixuan-ad-bot` → 開「完整控制」→ 儲存

## F. 產生金鑰（最關鍵的一步）

1. 系統使用者頁面 → 「產生新權杖」
2. 選 App：`kaixuan-ad-bot`
3. 權杖效期：選「**永不過期**」（never expire）
4. 勾權限（缺一不可）：
   - `pages_show_list`
   - `pages_read_engagement`
   - `pages_manage_posts`
   - `business_management`
5. 產生 → **馬上複製**（只顯示這一次！）→ 先貼到記事本暫存

## G. 換成「粉專權杖」（發文真正要用的）

**目的**：F 拿到的是系統使用者權杖，發文要用粉專自己的權杖，用 F 的權杖換一把。因為 F 永不過期，換出來的這把也永久有效。

1. 先拿粉專 ID：企業管理平台 → 帳號 → 粉絲專頁，名稱旁邊那串數字就是；記下來，這就是 `FB_PAGE_ID`
2. 瀏覽器開（把兩個值代進去）：
   ```
   https://graph.facebook.com/v21.0/{FB_PAGE_ID}?fields=access_token&access_token={F拿到的權杖}
   ```
3. 回傳 JSON 裡的 `access_token` 就是**粉專權杖** = `FB_PAGE_TOKEN`，複製暫存

## H. 驗證 + 存進 n8n

1. 驗證（瀏覽器開）：
   ```
   https://graph.facebook.com/v21.0/{FB_PAGE_ID}?fields=name&access_token={FB_PAGE_TOKEN}
   ```
   回傳粉專名稱 = 整條路通了 🎉
2. 進 n8n → Credentials → 新增「Header Auth」：
   - 名稱：`FB Page Token`
   - Header Name：`Authorization`
   - Header Value：`Bearer {FB_PAGE_TOKEN}`
3. **刪掉記事本暫存**。權杖等於粉專的鑰匙：不進 git、不貼群組、不留在桌面

## 常見卡點

| 症狀 | 解法 |
|---|---|
| 建 App 時沒有「商家」選項 | 選最接近「商業/企業用途」的類型即可，重點是能綁企業管理平台 |
| 權限清單找不到 `pages_manage_posts` | 確認 E 步驟有把 App 和粉專都指派給系統使用者，回頭補 |
| G 步驟回傳錯誤 | 九成是 F 的權限沒勾齊，重新產生一次權杖（舊的作廢沒關係） |
| 發文測試回 `(#200) permission denied` | 粉專資產的「完整控制」沒開，回 E 檢查 |
